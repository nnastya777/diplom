# Система учета рабочего времени сотрудников школы г. Вологды

Комплексная информационная система для автоматизированного учета рабочего времени, управления отпусками и формирования отчетности для образовательного учреждения.

## Основные возможности

### Для сотрудников
- ✅ Отметка времени прихода и ухода
- ✅ Просмотр статистики рабочего времени
- ✅ Подача заявок на отпуск и больничные
- ✅ Управление личным профилем

### Для руководителей
- ✅ Мониторинг посещаемости отдела
- ✅ Одобрение/отклонение заявок на отпуск
- ✅ Просмотр статистики команды
- ✅ Контроль опозданий и ранних уходов

### Для администраторов
- ✅ Управление пользователями и отделами
- ✅ Системные настройки
- ✅ Полный доступ ко всем данным
- ✅ Формирование сводных отчетов

### Система отчетности
- 📊 Отчет по посещаемости
- ⏰ Отчет по рабочим часам
- 📅 Отчет по отпускам и больничным
- 💰 Расчет заработной платы
- 📥 Экспорт в CSV, Excel, JSON

## Технологический стек

- **Frontend**: Next.js 16, React 19, TypeScript
- **UI**: Bootstrap 5, Tailwind CSS, shadcn/ui
- **База данных**: Supabase (PostgreSQL)
- **Аутентификация**: Supabase Auth с ролевой моделью
- **Деплой**: Vercel

### База данных

Схема состоит из следующих таблиц:

1. **profiles** - Профили пользователей
2. **attendance** - Учет посещаемости (приход/уход)
3. **work_hours** - Учет рабочих часов
4. **absences** - Отпуска, больничные, командировки
5. **reports** - Сохраненные отчеты
6. **notifications** - Системные уведомления

Все таблицы защищены Row Level Security (RLS) с настроенными политиками доступа.

### Ролевая модель

Система поддерживает три роли:

- **employee** (сотрудник) - базовый доступ к личному кабинету
- **manager** (руководитель) - доступ к управлению отделом
- **admin** (администратор) - полный доступ ко всем функциям

## Установка и запуск

### Предварительные требования

- Node.js 18+ 
- npm/pnpm/yarn
- Аккаунт Supabase

### Шаги установки

1. Клонируйте репозиторий:
\`\`\`bash
git clone <repository-url>
cd employee-time-tracking
\`\`\`

2. Установите зависимости:
\`\`\`bash
npm install
\`\`\`

3. Настройте переменные окружения:

Система автоматически использует переменные окружения из Vercel/Supabase интеграции:
- `SUPABASE_URL`
- `SUPABASE_ANON_KEY`
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

4. Выполните миграции базы данных:

В директории `scripts/` находятся SQL файлы для создания схемы БД. Выполните их в порядке нумерации:
- 001_create_users_and_profiles.sql
- 002_create_attendance_table.sql
- 003_create_work_hours_table.sql
- 004_create_absences_table.sql
- 005_create_reports_table.sql
- 006_create_notifications_table.sql
- 007_create_helper_functions.sql

5. Запустите проект:
\`\`\`bash
npm run dev
\`\`\`

Откройте [http://localhost:3000](http://localhost:3000) в браузере.

## Безопасность

### Аутентификация
- Email/Password через Supabase Auth
- Подтверждение email при регистрации
- Защита всех маршрутов через middleware

### Авторизация
- Row Level Security (RLS) на уровне БД
- Ролевая проверка на уровне приложения
- Политики доступа для каждой таблицы

### Защита данных
- HTTPS для всех соединений
- Хеширование паролей через Supabase
- Защита от SQL-инъекций через Supabase SDK

## Производительность

- Server-Side Rendering (SSR) для быстрой загрузки
- Оптимизация изображений через Next.js Image
- Кэширование на уровне Supabase
- Lazy loading компонентов

## Интеграция с системами расчета ЗП

Система предоставляет API для экспорта данных в форматах:
- CSV - для загрузки в Excel
- JSON - для программной интеграции
- Excel - прямой экспорт с форматированием

Данные включают:
- Отработанные часы (обычные и сверхурочные)
- Дни отсутствия с причинами
- Расчетные суммы на основе ставок
